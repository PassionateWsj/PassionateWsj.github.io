Java17-IO流

发表于 2018-01-10

一、介绍什么是IO流

  1. （IN-OUT STREAM），进来出去的刘。对数据流动性的表达，通过IO流的操作可以实现数据的录入和读取
  2. 可以与水进行类比

    二、特点

  • IO的In和Out都是以内存作为参照，如果向内存中写入，称为Out；读取称为In

三、IO流的分类

  1. 根据方向：输入流（In），输出流（Out）
  2. 根据类型分列：字节流(byte)通用性强(适合任何文件类型，图片，音频…)，字符流(char)（适合文本文件类型）
  3. 根据使用分类：节点流（可以直接使用的流），包装流（ 不能直接使用，包装完以后才能使用）

四、具体介绍流的使用

  • IO流(框架)的分类
    • InputStream–字节输入流
    • OutputStream–字节输出流
    • Reader–字符输入流
    • Writer–字符输出流
      • 比如：
        FileOutputStream 文件字节输出流
        FileReader 文件字符输入流
        PrintWriter 打印字符输出流

五、文件流比较重要的，用的特别多（在finally中关闭流 .close()）

  1. FileInputStream，FileOutputStream
  2. FileReader，FileWriter

总结：

  1. FileOutputStream提供核心的方法write()
    • write(int b) //b就每一字节 一个一个写入（写入中文的时候无法实现）
    • write(byte[] b) //写入整个byte数组中所有的内容，使用String可以转换成byte[]，在网络传输过程中可以直接获取】
    • write(byte[] b , int offset , int len) //是对上一个方法的定制
      比如：byte[] b = {97，97，99，200，101，44} 使用write(b,3,2) –>{200，101}

补充：写入的时候可以设置追加模式，在new FileOutputStream可以填写第二个参数：填写true表示追加模式

  1. FileInputStream提供的核心方法read();
    • int read() //读取每一个字节，返回int类型（返回-1表示读取完毕）
    • int read(byte[] buffer) //byte[]目的，将读取的每一个byte存放到buffer数组中，返回int表示数组中
      read()是一个阻塞线程的方法

六、缓冲流（特点：包装流，不能直接使用，要在节点流的基础上使用）使用频率：中高

  • 关键字：Buffered
  • 四种类型：BufferedInputStream，BufferedOutputStream，BufferredReader，BufferedWriter

七、内存流（特点：节点流，将内容存放到内存中或者从内存中读取内容）使用频率：中

  • 和文件流相比，他是对内存进行操作
  • 名称：ByteArrayOutputStream和ByteArrayInputStream
    使用的方法还是write和read

八、数据流（特点：包装流，可以写入或读取基本数据类型，int,boolean.double,字符串也可以）使用频率：中

DataInputStream
DateOutputStream

九、打印流（包装流，可以通过打印的方式写入或者读取内容）使用频率：低

PrintStream子类PrintWriter 写的方法.print() .println()

十、对象流（特点：包装流，可以将对象写入或读取，针对文件流进行对象的读写）使用频率：高

ObjectOutputStream,ObjectInputStream 写入的内容是Object实例化对象

十一、转换流（使用在节点流和包装流之间，将字节流转换成字符流）使用频率：低

十二、随机流

Java08-抽象类和接口

发表于 2018-01-10

一、抽象类

定义：这个类是抽象的，抽象的东西就是不完整的东西，他不可能被直接创建，直接去new是不可以的
表达抽象类的关键字：abstract 表示抽象 可以形容类，也可以形容方法
比如：
  • 总结抽象类特点：
    1. 抽象类里可以没有抽象方法
    2. 如果一个类里存在抽象方法，这个类一定是抽象类
    3. 抽象类必须被继承以后才能实例化

在面向对象的概念中，所有的对象都是通过类来描绘的，但是反过来，并不是所有的类都是用来描绘对象的，如果一个类中没有包含足够的信息来描绘一个具体的对象，这样的类就是抽象类。

抽象类除了不能实例化对象之外，类的其它功能依然存在，成员变量、成员方法和构造方法的访问方式和普通类一样。
由于抽象类不能实例化对象，所以抽象类必须被继承，才能被使用。也是因为这个原因，通常在设计阶段决定要不要设计抽象类。
父类包含了子类集合的常见的方法，但是由于父类本身是抽象的，所以不能使用这些方法。

二、接口

定义：表示同意的一种行为规范
比如：值钱的，美味的，能造东西的
Java中认为只要是able，ability结尾的可以定义为接口，形成一种规范
表达接口的关键字：interface
比如：
interface Valuable{} 对比 class Valueablt{}

  • 接口里可以写哪些东西？
    1. 只能写常量（static final 来表示）
    2. 只能写抽象方法

3. 接口中的常量和抽象方法的访问修饰符只能是public

特点：等待其他的类来实现（implements）该接口（interface），完善里面的规则
比如：大熊猫值钱的，要实现Valuable接口,关键字implements实现接口

记忆的时候class和interface是一块的的，然后extends和implements是一块的

  • 请描述下抽象类和接口的区别：
    1. 抽象类是被继承的，接口是被实现的
      Java规定继承只能是单继承，Java规定实现可以是多实现，在每个接口直接用逗号隔开就可以了
    2. 抽象类里可以没有抽象的方法，但是接口不行，要么不写方法，要写方法都是抽象方法
    3. 抽象类里可以定义变量，接口内部只能定义常量，没有变量的说法
    4. 抽象类内部访问修饰除了private其他都可以定义，接口只能用public进行访问修饰
    5. 接口编程灵活性更加强，是比抽象类更抽象的一个东西

接口（英文：Interface），在JAVA编程语言中是一个抽象类型，是抽象方法的集合，接口通常以interface来声明。一个类通过继承接口的方式，从而来继承接口的抽象方法。
接口并不是类，编写接口的方式和类很相似，但是它们属于不同的概念。类描述对象的属性和方法。接口则包含类要实现的方法。
除非实现接口的类是抽象类，否则该类要定义接口中的所有方法。
接口无法被实例化，但是可以被实现。一个实现接口的类，必须实现接口内所描述的所有方法，否则就必须声明为抽象类。另外，在Java中，接口类型可用来声明一个变量，他们可以成为一个空指针，或是被绑定在一个以此接口实现的对象。

  • 接口与类相似点：

    • 一个接口可以有多个方法。
    • 接口文件保存在.java结尾的文件中，文件名使用接口名。
    • 接口的字节码文件保存在.class结尾的文件中。
    • 接口相应的字节码文件必须在与包名称相匹配的目录结构中。

  • 接口与类的区别：

    • 接口不能用于实例化对象。
    • 接口没有构造方法。
    • 接口中所有的方法必须是抽象方法。
    • 接口不能包含成员变量，除了static和final变量。
    • 接口不是被类继承了，而是要被类实现。
    • 接口支持多重继承。

Java15-Set集合

发表于 2018-01-10

Set集合特点

继承于 Collection，是一个不包含重复元素的集合。
在Set集合里不会有重复的元素，遍历集合的时候，获取的元素不是按照存取的顺序获取的，是无序的
因为没有下标这种说法，普通的for循环无法便利Set集合，所以只能用迭代器或者foreach
所有超级接口：
Collection, Iterable
所有已知实现类：
AbstractSet, ConcurrentSkipListSet, CopyOnWriteArraySet, EnumSet, HashSet, JobStateReasons, LinkedHashSet, TreeSet
比较常用的Set的实现类：HashSet，TreeSet

HashSet（按照哈希表的散列规则进行数据的存放）

继承了AbstractSet，并且使用一个哈希表。
定义类的时候，必需要重写Obje提供的2个方法hashCode()，equals()（除了基本数据类型类和String类）
这样才能保证Set集合的元素不重复
直接通过equals进行对象比较，在Set集合里效率太低，需要Hash散列算法进行辅助

总结HashCode的说法（散列算法保证元素尽量在hash表分配均匀）
a.相同的对象，hashCode一定相同
b.不同的对象，hashCode可能相同
c.hashCode不同的，对象一定不同

TreeSet（通过Comparable和Comparator来排序的同时判断是否重复）（一定要有一种（些）顺序排序）

继承于AbstractSet，使用元素的自然顺序对元素进行排序.
添加，删除的操作流程都是一样的
可以不需要重写hashCode和equals，必须要重写Comparable接口或者Comparator接口，没有比较的属性直接忽略（都视为相同）
特点：获取的数据按照排序的规则进行实际的获取，不是无序的

  • 总结1：Comparable和Comparator的区别

    • 共同点：可以自己定义规则进行对象的比较
    • 不同点：
      • Comparable重写的是compareTo方法，只能采用一套比较规则
      • Comparator重写的是compare方法，只要定义类实现Comparator，可以写多个比较器，定义不同的比较规则

  • 总结2：Set中操作数据add，remove所有和下标索引有关的方法都是没有的
