  1. 自定义 View 要点：
    • 如果不起作用，需要关闭硬件加速
      • setLayerType(View.LAYER_TYPE_SOFTWARE, null);
    • setXfermode(Xfermode m)：设置图形重叠时的处理方式
    • 通过paint. setXfermode(new PorterDuffXfermode(PorterDuff.Mode.XXX))来设置
      1. PorterDuff.Mode.CLEAR：所绘制不会提交到画布上。
      2. PorterDuff.Mode.SRC：显示上层绘制图片
      3. PorterDuff.Mode.DST：显示下层绘制图片
      4. PorterDuff.Mode.SRC_OVER：正常绘制显示，上下层绘制叠盖。
      5. PorterDuff.Mode.DST_OVER：上下层都显示。下层居上显示。
      6. PorterDuff.Mode.SRC_IN：取两层绘制交集。显示上层。
      7. PorterDuff.Mode.DST_IN：取两层绘制交集。显示下层。
      8. PorterDuff.Mode.SRC_OUT：上层绘制非交集部分。
      9. PorterDuff.Mode.DST_OUT：取下层绘制非交集部分。
      10. PorterDuff.Mode.SRC_ATOP：取下层非交集部分与上层交集部分
      11. PorterDuff.Mode.DST_ATOP：取上层非交集部分与下层交集部分
      12. PorterDuff.Mode.XOR：异或：去除两图层交集部分
      13. PorterDuff.Mode.DARKEN：取两图层全部区域，交集部分颜色加深
      14. PorterDuff.Mode.LIGHTEN：取两图层全部，点亮交集部分颜色
      15. PorterDuff.Mode.MULTIPLY：取两图层交集部分叠加后颜色
      16. PorterDuff.Mode.SCREEN：取两图层全部区域，交集部分变为透明色

  3. ScrollView 嵌套 ListView 滚动问题
    • 问题 ：在 ListView 的 Adapter NotifyDataChanged() 方法时，Scroll 会滚动到 ListView 的位置
    • 是因为 ListView 刷新数据之后 ，调用了 ScrollView 的 onLayout 方法，这个方法会根据 child 的聚焦状态来判断是否让 ScrollView 滚动到 child view 的位置
    • 解决：给 child view 添加 setFocusable(false)，初步解决问题

  9. SharedPreferences.Editor 提交方法 commit() 和 apply() 区别

    • commit() : 同步，有返回结果
    • apply() : 异步，无返回结果
    • commit和apply虽然都是原子性操作，但是原子的操作不同，commit是原子提交到数据库，所以从提交数据到存在Disk中都是同步过程，中间不可打断。
    • 而apply方法的原子操作是原子提交的内存中，而非数据库，所以在提交到内存中时不可打断，之后再异步提交数据到数据库中，因此也不会有相应的返回值。
    • 所有commit提交是同步过程，效率会比apply异步提交的速度慢，但是apply没有返回值，永远无法知道存储是否失败。
    • 在不关心提交结果是否成功的情况下，优先考虑apply方法。

  12. Kotlin 小知识点
    • 修饰词（可以用在类，对象，接口，构造函数，属性以及属性的set()中。）
      • private : 只有在本源文件中可见 , 该属性不能被子类访问
      • protected : 该属性能被子类访问
      • internal : 同一模块下可见
      • public(default) : 到处都可以被访问
    • 静态方法
      • 工具类（全是静态方法的情况）
        • 将 class 类名 改为 object 类名 即可
      • 普通静态方法
        • 将方法用 companion object { } 包裹即可
    • val 声明的成员变量 : 在 init 中实例化就可以不赋初始值
  13. 让你的 Kotlin 代码远离 !!
    • !! : 表示潜在未处理的 NullPointerException，消除 !! 能让代码更安全、更容易 debug 并且更干净
    • 6 种处理方式
      1. 用 val 而不是 var
      2. 使用 lateinit
      3. 使用 let 函数
      4. 创建全局函数来处理更复杂的情况
      5. 使用 Elvis 运算符
      6. 自定义崩溃信息
